# Cudy TR3000 新人完全手册（从连接到可用）

> 适用对象：第一次接触路由/OpenClash 的用户  
> 目标：下次你自己看这份文档，也能独立连上并完成基础配置

---

## 1. 先搞清楚你要做什么

你现在这台 Cudy 的作用是：
- 上游连家庭网络（例如华为子母路由）
- 下游给你的设备上网
- 用 OpenClash 做规则分流（国内直连、国外代理）

一句话：它是你的“中间代理路由”。

---

## 2. 硬件连接（推荐长期方案）

接线方式：
- 上级路由（华为）的 `LAN` 口 -> Cudy 的 `WAN` 口

你的设备连接方式（二选一）：
- 连 Cudy 发出的 Wi-Fi
- 或者网线插 Cudy 的 LAN 口

> 备注：上游无线中继也能用，但速度可能下降。稳定优先时，先按上面的有线上联方式。

---

## 3. 如何进入管理页面

### 3.1 已连上 Cudy 网络时
浏览器打开：

```text
http://192.168.1.1
```

### 3.2 找不到管理地址时（Mac 终端）
执行：

```bash
for ip in 192.168.1.1 192.168.2.1 192.168.3.1 192.168.8.1 192.168.10.1; do echo "== $ip =="; curl -sS -I --max-time 2 http://$ip | sed -n '1p'; done
```

看到返回 `HTTP/1.1 200` 的地址，就是管理入口。

---

## 4. 首次必须做的安全设置

进入后台后，先做这几件事：

1. 改管理密码（系统/管理权页面）
2. 设置 Wi-Fi 密码（2.4G 和 5G 都要设）
3. 无线加密选 `WPA2-PSK` 或 `WPA2/WPA3`
4. 关闭 WPS
5. SSH 只保留 `LAN` 访问，不开 WAN 暴露

> 你已经完成了大部分，上面是给以后复查用。

---

## 5. OpenClash 基础可用配置（最稳模板）

在 OpenClash 运行状态页，保持如下：

- 运行模式：`Fake-IP`
- 代理模式：`规则`
- DNS 代理：`启用`
- 域名嗅探：`启用`
- 区域绕过：`大陆`

然后点一次：
- 清理 DNS 缓存
- 重启防火墙

最后看“访问检查”：
- 百度正常
- GitHub / YouTube 正常

满足这三个，说明基础可用。

## 6. 订阅与节点（你最关心）

### 6.1 添加订阅
OpenClash -> 配置订阅 -> 添加配置 -> 订阅链接 -> 添加配置。

### 6.2 更新订阅
回到运行状态页，在配置文件区点“圆圈刷新”按钮更新。

### 6.3 切节点
点 `Metacubexd` 面板，进代理组切换节点。  
切完看当前节点是否变化，再刷新确认。

## 8. 常见问题快查

### Q1: `opkg update` 失败，提示证书未生效
原因：路由器系统时间错误。  
处理：先校时，再 `opkg update`。

### Q2: OpenClash 安装了但不运行
原因：通常是内核/核心未就绪。  
处理：在插件设置里确认 Meta 内核可用，再应用配置。

### Q3: 点了节点感觉没保存
处理：看代理组“当前节点”是否变化，再刷新一次确认。

### Q4: 接到上级路由后找不到 192.168.1.1
原因：上下级 NAT 导致管理面隔离。  
处理：先连回 Cudy 网络再进后台，或用扫描命令找管理地址。
