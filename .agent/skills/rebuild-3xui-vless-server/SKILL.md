# Rebuild 3x-ui VLESS Server

## Goal

Build the same practical outcome as the reference server: Docker-managed 3x-ui, one public VLESS + REALITY node, and one root-level Markdown document containing all client configuration details.

Default target:

- 3x-ui image: `ghcr.io/mhsanaei/3x-ui:latest`
- Container name: `3x-ui`
- Data directory: `/opt/3x-ui`
- Node protocol: `vless`
- Node transport/security: `tcp` + `reality`
- Node port: `443`
- Subscription/document port: `2096` when enabled by 3x-ui
- Output document: `/root/vless-reality-node-info.md`

## Workflow

1. Inspect the server first.
   Run `scripts/inspect_server.sh` and review CPU, memory, disk, OS, public IP, Docker state, listening ports, firewall hints, and existing 3x-ui containers.

2. Ask one question at a time only when needed.
   Prefer defaults on fresh VPS instances. Ask before replacing an existing `3x-ui` container, reusing an occupied node port, changing firewall rules, or overwriting an existing `/root/vless-reality-node-info.md`.

3. Deploy with the bootstrap script.
   Run `scripts/bootstrap_3xui_vless.sh` as root. Let the script install required packages, install Docker if missing, deploy 3x-ui, create the VLESS REALITY inbound, restart the container, and write the node document.

4. Verify the result.
   Confirm `docker ps` shows `3x-ui`, `ss -tulpn` shows the VLESS port, and `/root/vless-reality-node-info.md` exists with mode `600`.

5. Give the user the shortest usable next step.
   Tell the user to open/download `/root/vless-reality-node-info.md` and import the `vless://` URI into their client. Include the SSH tunnel command for local-only panel access if present in the document.

## Scripts

- `scripts/inspect_server.sh`: Read-only server inventory. Safe to run before every deployment.
- `scripts/bootstrap_3xui_vless.sh`: Interactive/root deployment script. Installs dependencies, starts Docker, deploys 3x-ui, writes the inbound to 3x-ui SQLite storage, and creates `/root/vless-reality-node-info.md`.

Run from the skill directory:

```bash
sudo bash scripts/inspect_server.sh
sudo bash scripts/bootstrap_3xui_vless.sh
```

## Interaction Rules

- Use Chinese for user-facing prompts if the user is Chinese.
- Do not dump a large questionnaire. Ask the next blocking question only.
- Treat existing services as user-owned. Do not delete containers, files, firewall rules, or existing inbounds without explicit confirmation.
- Do not hard-code secrets from another server. Generate a fresh UUID, REALITY private key, public key, short ID, panel path, and subscription client ID on each new VPS.
- If the user requests exact cloning, explain that IP, domain, keys, and UUID should be regenerated on a new VPS; only the deployment shape and client effect should be duplicated.

## Defaults

Use these defaults unless the user specifies otherwise:

- Public address: detected public IPv4; ask if detection fails.
- Remark: `vless-reality-<hostname>`
- VLESS port: `443`
- Flow: `xtls-rprx-vision`
- Fingerprint: `chrome`
- REALITY target: `www.microsoft.com:443`
- REALITY SNI: `www.microsoft.com`
- SpiderX: `/`
- 3x-ui panel listen address: `127.0.0.1`
- 3x-ui panel port: random high localhost port generated by the panel, or `36532` if the script initializes settings
- 3x-ui subscription port: `2096`
- 3x-ui subscription path: `/trade/`
